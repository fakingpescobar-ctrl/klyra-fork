# klyra-fork

Форк проекта [tg-prplx/klyra](https://github.com/tg-prplx/klyra) — агентный CLI для программирования на Go.

## Что добавлено

### 1. Параллельные вызовы инструментов
Если LLM запрашивает несколько инструментов за один шаг (например, прочитать три файла), они теперь выполняются конкурентно через горутины. Проверки на дубли и апрув по-прежнему последовательны, результаты добавляются в контекст в оригинальном порядке.

### 2. Retry с экспоненциальным backoff для LLM
При ошибках 429, 502, 503, 504, rate limit или разрыве соединения агент автоматически повторяет запрос до 3 раз с задержками 1с → 2с → 4с. Уважает отмену контекста.

### 3. Обрезка длинных выводов инструментов
Вывод каждого инструмента обрезается до 32 КБ перед записью в контекст. Это защищает окно контекста от переполнения при больших выводах bash или файловых операций.

### 4. Провайдер Ollama
Новый `pkg/llm/ollama.go` — тонкая обёртка над OpenAI-compatible API. Подключается к локальному серверу Ollama по умолчанию на `localhost:11434`. Поддерживает переменную окружения `OLLAMA_HOST`.

```go
provider, err := llm.NewOllamaProviderFromEnv()
// или явно:
provider, err := llm.NewOllamaProvider("http://localhost:11434/v1")
```

### 5. Суб-агенты
Новый инструмент `sub_agent` позволяет агенту делегировать изолированные подзадачи дочерним агентам. Дочерний агент получает собственный контекст, свои вызовы инструментов и возвращает результат как вывод инструмента. Рекурсия заблокирована.

```go
cfg := agent.Config{
    // ...
    SubAgentFactory: agent.DefaultSubAgentFactory(cfg),
}
```

### 6. Калибровка подсчёта токенов
`BudgetedWindow` теперь сравнивает оценочное количество токенов с реальным из ответа API и корректирует внутренний коэффициент через экспоненциальное скользящее среднее (α=0.3). Это предотвращает переполнение контекстного окна при систематическом занижении оценки.

### 7. Корректная обработка Ctrl+C
Проверка `ctx.Done()` добавлена в начале каждого шага основного цикла, перед запуском горутин и после их завершения. Агент завершается немедленно при отмене контекста, не дожидаясь следующего шага.

### 8. Структурированные логи (slog)
В `Config` добавлено поле `Logger *slog.Logger` (по умолчанию `slog.Default()`). Агент логирует вызовы инструментов, ошибки, политику безопасности, retry и использование токенов в структурированном формате — удобно для CI/CD, мониторинга и отладки.

```go
import "log/slog"

logger := slog.New(slog.NewJSONHandler(os.Stderr, nil))
cfg := agent.Config{
    Logger: logger,
    // ...
}
```

## Сборка

Требования: Go 1.21+, gcc (для go-tree-sitter).

```bash
git clone https://github.com/fakingpescobar-ctrl/klyra-fork
cd klyra-fork
go build -o klyra ./cmd/klyra/
```

## Изменённые файлы

| Файл | Что изменилось |
|------|----------------|
| `pkg/agent/agent.go` | Параллельные tool calls, retry, обрезка выводов, ctx checks, slog |
| `pkg/agent/subagent.go` | Новый файл: инструмент `sub_agent` и `SubAgentFactory` |
| `pkg/llm/ollama.go` | Новый файл: провайдер Ollama |
| `pkg/context/window.go` | Калибровка токенов через `CalibrateFrom()` |
| `pkg/tools/registry.go` | Регистрация `sub_agent` в реестре инструментов |
